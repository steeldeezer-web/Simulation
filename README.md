# Симуляция 2D мира с травоядными и хищниками

## Описание проекта
Этот проект представляет собой пошаговую симуляцию 2D мира, где живут травоядные и хищники. Мир состоит из клеток в виде матрицы NxM, на которых могут располагаться живые существа, ресурсы (трава) и статичные объекты (камни, деревья). Каждая клетка может содержать не более одного объекта или существа.

## Ключевые особенности
- Используется объектно-ориентированный подход с иерархией классов.
- Реализован алгоритм поиска пути в ширину (BFS) для нахождения кратчайшего пути к цели.
- Существа принимают решения о перемещении и взаимодействиях с окружением.
- Консольный рендеринг текущего состояния мира.
- Управление симуляцией через класс Simulation с подсчетом ходов.

## Структура проекта

### Класс Coordinate
- Хранит координаты клетки на карте.
- Поддерживает корректное сравнение и хэширование для коллекций.
- Переопределяет toString() для удобства вывода.

### Абстрактный класс Entity
- Базовый класс для всех объектов, присутствующих на карте.
- Содержит уникальный идентификатор UUID и позицию.

### Классы Grass, Rock, Tree
- Представляют ресурсы и препятствия.
- Наследуются от Entity.

### Абстрактный класс Creature
- Определяет основные свойства живых существ: скорость, здоровье, имя и ссылку на карту.
- Содержит абстрактный метод makeMove() для реализации поведения каждого существа.
- Методы проверки жизни, уменьшения здоровья.

### Классы Herbivore и Predator
- Реализуют методы makeMove() с логикой поиска целей с помощью BFS.
- Herbivore ищет траву, Predator — травоядных.
- Predator наносит урон травоядным при встрече.

### Класс WorldMap
- Хранит карту и размещение объектов.
- Методы проверки границ, свободных клеток.
- Управление добавлением, удалением и перемещением объектов.

### Класс PathFinder
- Реализует BFS для поиска кратчайшего пути к ближайшему объекту заданного типа.
- Восстанавливает список координат пути.

### Класс Renderer
- Отвечает за консольный вывод карты.
- Визуализирует объекты с помощью однобуквенных обозначений.

### Класс Simulation
- Управляет состоянием мира и списком существ.
- Контролирует ход симуляции, обновление карты и рендеринг.
- Методы запуска, паузы и одиночного хода.

### Класс Main
- Точка входа.
- Создаёт объекты, запускает симуляцию.

## Запуск и использование
1. Скомпилируйте все классы.
2. Запустите класс Main.
3. В консоли отобразятся 20 ходов симуляции с позициями объектов.
4. Можно менять начальные параметры и расширять функционал.
